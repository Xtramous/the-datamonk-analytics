🚀 Apache Spark Setup - Complete Overview

Understanding what "Spark Setup" means, what gets installed, and how it all works together

1️⃣ What Does "Setting Up Spark" Mean?

Simple Definition:
Spark Setup = Installing and configuring Apache Spark + its dependencies so you can write and run distributed data processing jobs on your local machine or cluster.

Setup Involves Three Layers:

1Prerequisites (Foundation)

  • Java JDK - Spark runs on Java Virtual Machine (JVM)
  • Environment Variables - Tell OS where to find Java and Spark
  • Package Manager - Homebrew (macOS) to install everything

2Spark Installation

  • Spark Binary - Download and extract Spark files
  • Configuration Files - Setup logging, memory, executor settings
  • PySpark (Optional) - Python API for Spark; requires Python 3

3Development Environment

  • Project Directory Structure - Where your code, data, and config live
  • IDE/Editor - VS Code or PyCharm for writing code
  • Testing & Debugging Tools - Spark Shell, Jupyter, logging
💡 Key Concept: Local vs Cluster Mode
When you "setup Spark locally," you're installing Spark to run in Local Mode = Spark runs on your single machine using multiple threads (not actual distributed). This simulates how Spark would work on a real cluster, but everything stays on your laptop.

2️⃣ Installation Order (What Gets Installed First?)

Java JDK → Environment Variables → Spark Binary → Python & PySpark → Project Setup
Order Component Purpose Size Time
1️⃣ First Java JDK 8 or 11 Spark's runtime environment. Spark code compiles to Java bytecode ~300MB 2-3 min
2️⃣ Second Environment Variables Tell OS where Java & Spark are installed. Added to ~/.zprofile N/A 1 min
3️⃣ Third Spark Binary The actual Spark framework (~400MB when extracted) ~180MB 2-3 min
4️⃣ Fourth Python 3 + Jupyter For writing PySpark code instead of Scala. Development experience ~500MB 5 min
5️⃣ Fifth Project Directory Where your code, data, config files live Varies 1 min
⚠️ Common Mistake:
People try to install Spark before Java. This won't work! Spark needs Java to run. Always: Java → Environment Variables → Spark.

After Setup, Your Disk Will Look Like This:

Cellar/

config/ ← spark-defaults.conf, log4j.properties
💡 Environment Variables Explained
Environment variables are instructions in ~/.zprofile that tell your shell (Terminal):
  • JAVA_HOME=/path/to/java → "When running Java, use this version"
  • SPARK_HOME=/path/to/spark → "When running Spark, use this directory"
  • PATH=$PATH:$SPARK_HOME/bin → "Add Spark to commands you can run"
Without these, Terminal won't know where Java or Spark are!

5️⃣ How Spark Runs on Your Machine (Local Mode Explained)

Spark Architecture in Local Mode:

Your Python Script → Driver Process (on your machine) → Executor Threads (on your machine) → Results
Component Location What It Does In Your Setup
Driver Your machine Reads your Python code, creates Spark plan, manages execution Main Python process
Executors Your machine (multiple threads) Processes data in parallel; actual computation happens here Multiple CPU threads
Spark Context Driver Entry point to Spark cluster; tells Spark how to access data and resources SparkSession.builder.getOrCreate()
Memory Your machine's RAM Stores DataFrames, intermediate results, shuffle data Default ~1GB; configurable
Simple Example: How Your Code Executes
  1. You run: python my_spark_job.py
  2. Python starts; creates Spark session (Driver starts)
  3. Your code creates a DataFrame: df = spark.read.csv("data.csv")
  4. You call an action: df.show() (this triggers execution)
  5. Driver creates execution plan → breaks into stages/tasks
  6. Executor threads work in parallel on different partitions
  7. Results come back to Driver; printed to console
  8. Job finishes; Spark session closes
⚠️ Key Spark Concept: Lazy Evaluation
When you do df.filter(...).select(...), Spark doesn't execute immediately. It just builds a plan. Execution only happens when you call an action like .show(), .collect(), or .write(). This is why learning the DAG and Catalyst is important!

6️⃣ How You'll Run Your Code (Three Ways)

Method 1: Spark Shell (Interactive)

$ spark-shell # Launches Scala REPL # Type Spark code directly; see results immediately scala> val df = spark.read.csv("data.csv", header=true) scala> df.show() # Great for exploration, understanding DAG via UI

Best for: Learning, debugging, quick exploration

Method 2: Python Scripts

# File: scripts/my_job.py from pyspark.sql import SparkSession spark = SparkSession.builder \ .appName("MyJob") \ .getOrCreate() df = spark.read.csv("data.csv", header=True) df.show() # Run from command line: $ spark-submit scripts/my_job.py # Or: python scripts/my_job.py (requires SPARK_HOME in PATH)

Best for: Production jobs, reproducible code, logging, error handling

Method 3: Jupyter Notebooks

# Inside Jupyter notebook cell from pyspark.sql import SparkSession spark = SparkSession.builder \ .appName("Notebook") \ .getOrCreate() df = spark.read.csv("data.csv", header=True) df.show() # Results display below cell

Best for: Data exploration, visualization, documentation, learning

💡 Which Method for L5 Learning?
  • Weeks 1-4 (Spark Internals): Use Spark Shell + Jupyter to analyze EXPLAIN plans and DAGs via Spark UI
  • Weeks 5-8 (Streaming): Use Python scripts with logging to handle checkpoints, recovery
  • Weeks 9-12 (System Design): Use Python scripts + documentation for production patterns

7️⃣ Spark UI: Visualizing Your Jobs (Critical for Learning)

What is Spark UI?
Spark UI = Web dashboard showing job execution in real-time
  • See DAG (Directed Acyclic Graph) visualized
  • Watch stages, tasks, timing, data movement
  • Understand where time is spent
  • Identify bottlenecks (shuffles, memory issues)

Accessing Spark UI:

Step 1: Run any Spark job (shell, script, or notebook)
Step 2: While job is running (or just finished), open browser
Step 3: Go to: http://localhost:4040
Step 4: Click on "Jobs" tab to see DAG visualization
✅ What You'll See:
  • Job → Stages (colored boxes)
  • Stages → Tasks (parallelized work)
  • Shuffle operations (data movement)
  • Execution time per stage
  • Executor memory usage
⚠️ Spark UI Tips
  • UI only works while Spark context is alive (job running or just finished)
  • After job completes, you have ~30 seconds before UI disappears
  • For analysis, take screenshots immediately!
  • Or enable persistent history server (advanced topic)

8️⃣ Configuration Files (The Config Layer)

Three Key Config Files:

1. log4j.properties (Logging)

Location: $SPARK_HOME/conf/log4j.properties

# Controls what messages Spark prints to console/logs log4j.rootCategory=WARN, console # Change WARN to INFO to see more details (useful for debugging)

2. spark-defaults.conf (Default Settings)

Location: $SPARK_HOME/conf/spark-defaults.conf

# Sets default values for all Spark jobs spark.driver.memory 2g spark.executor.memory 2g spark.executor.cores 4 spark.sql.shuffle.partitions 200

3. Your Job Config (Inside Python Code)

from pyspark.sql import SparkSession spark = SparkSession.builder \ .appName("MyJob") \ .config("spark.executor.memory", "2g") \ .config("spark.executor.cores", "4") \ .config("spark.sql.shuffle.partitions", "200") \ .getOrCreate()
💡 Config Priority Order
When you run a Spark job, these override each other:
  1. spark-submit --conf arguments (highest priority)
  2. In-code .config() calls
  3. spark-defaults.conf
  4. Spark's built-in defaults (lowest priority)
For L5 learning, you'll mostly use in-code config for experimentation.

# In output/week1_analysis.md Job Name: DAGExploration Stages: 2 (read → group by) Shuffle: Yes (stage 1 → 2) Tasks: 8 total (4 in each stage) Time: 2.3 seconds Observations: Filter and select are pushed into stage 0 (predicate pushdown)

🔟 Pre-Setup Checklist (Before Installing)

  • Machine Type: You have a Mac with M1/M2/M3 or Intel processor
  • Storage: At least 5GB free disk space (Java + Spark + dependencies)
  • Memory: At least 8GB RAM (Spark local mode needs ~2GB per default)
  • Terminal Access: Can open Terminal and run commands
  • Homebrew Installed: If not, we'll install it (macOS package manager)
  • Internet: Download packages (~1GB total)
  • Python 3 Installed: macOS comes with it, but might need to verify
  • Code Editor: VS Code or similar ready for editing Python files
    • ✅ After Setup You Should Be Able To:
    • Open Terminal and type: spark-shell → Scala REPL starts
    • Create a Python file, run: spark-submit my_script.py → Job executes
    • While running, open localhost:4040 → See Spark UI
    • Check Spark version: spark-submit --version
    • Check Java version: java -version
