📈 Performance SLAs

Real-time queries: <100ms (p99)
Analytical queries: <500ms (p99)
Cache hit rate: 70%
Event latency: <2min to dashboard
Hot data retention: 7 days (Redis)
Cold data retention: 2 years (Snowflake)

🔄 Key Decisions

Dual Storage Redis (fast) + Snowflake (scalable)
Dual Compute Flink (low-latency) + Spark (batch correctness)
Query APIs 3 paths for 3 access patterns
Consistency Eventual (5min lag) vs. synchronous writes
Schema Registry versioning + replay capability

💰 Cost & Scaling

Current (1000 metrics): ~$22K/mo
Kafka: $3K | Flink: $8K | Redis: $5K | Snowflake: $6K

Scale to 5000 metrics:
Shift 80% hot data → Snowflake, keep top 200 in Redis
Add intermediate hourly Spark layer
Linear cost growth to ~$35K/mo

⚡ Reliability

Exactly-once: Spark idempotent writes (upsert), Flink checkpointing
Replay: Kafka 7d retention, re-run Spark with date-range
Quality: Flink validation + dbt tests + reconciliation alerts
Backfill: Schema Registry versioning enables re-processing

🎯 Trade-offs

Dual storage: Complexity vs. meeting both latency SLAs
✓ Can't do <100ms with only Snowflake
Kafka vs Kinesis: Ops overhead vs. $7K/mo savings
✓ K8s auto-scaling + Schema Registry justify cost
Eventual consistency: 5min lag vs. synchronous writes complexity
✓ Nightly reconciliation catches discrepancies
Materialized views: Forecasting effort vs. 99% query speedup
✓ Top 50 metrics cover 80% of queries

📋 Interview Narrative

"The core challenge: real-time (<100ms) and historical (2yr retention) have conflicting requirements. I use dual storage—Redis for 7-day hot data, Snowflake for cold. Two compute engines: Flink streams to Redis (15s windows), Spark batch nightly for correctness. Three query APIs optimize for different clients. Most queries (70%) hit cache because users repeat the same questions.

Biggest trade-off: eventual consistency. Redis and Snowflake lag ~5 minutes. For revenue metrics, I run nightly reconciliation with >1% alerts. It's deliberate: buy latency by accepting eventual consistency.

At 100 users, 1000 metrics: ~$22K/mo. Scale to 5000 metrics: shift hot data to Snowflake, keep top 200 in Redis."
