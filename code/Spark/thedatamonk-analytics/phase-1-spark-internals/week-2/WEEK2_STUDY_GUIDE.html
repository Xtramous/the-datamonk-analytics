🔧 RDD Fundamentals

What is an RDD?

An RDD (Resilient Distributed Dataset) is the fundamental data structure of Apache Spark. It is an immutable, distributed collection of objects that can be processed in parallel.

Key Properties of RDDs:
  • Resilient: Can recover from node failure
  • Distributed: Data is split across multiple nodes
  • Immutable: Cannot be changed after creation
  • Lazy Evaluated: Operations are not executed immediately
  • Typed: Each element has a type

RDD Lineage & DAG

Every RDD knows how it was derived from other RDDs through lineage tracking.

RDD Creation Lineage: Original Data │ (flatMap) ▼ RDD1: Flattened Data ┌─────────────────────┐ │ Partition 0: [...] │ │ Partition 1: [...] │ │ Partition 2: [...] │ └─────────────────────┘ │ (filter) ▼ RDD2: Filtered Data ┌─────────────────────┐ │ Partition 0: [...] │ │ Partition 1: [...] │ │ Partition 2: [...] │ └─────────────────────┘ │ (map) ▼ RDD3: Mapped Data ┌─────────────────────┐ │ Partition 0: [...] │ │ Partition 1: [...] │ │ Partition 2: [...] │ └─────────────────────┘

⚡ Transformations vs Actions

Lazy Evaluation Model

Spark uses lazy evaluation: transformations are not computed immediately. Computation happens only when an action is called.

from pyspark import SparkContext

sc = SparkContext("local", "Lazy Evaluation")

# Transformation 1: CREATE (no execution)
rdd1 = sc.parallelize([1, 2, 3, 4, 5])

# Transformation 2: MAP (no execution)
rdd2 = rdd1.map(lambda x: x * 2)

# Transformation 3: FILTER (no execution)
rdd3 = rdd2.filter(lambda x: x > 5)

# ACTION: COLLECT (triggers execution!)
result = rdd3.collect()
print(result)  # [6, 8, 10]
# NOW all transformations execute in pipeline
